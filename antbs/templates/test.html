Introduction

The Sound Blaster series of soundcards have been around Personal Computers for a long time... in fact? The first soundblaster dates back to 1989... a year before Windows 3.1 came out. They were one of the few soundcards at the time that could perform the complex sound processing needed to play games with the sounds that the developers aimed for. Nowadays the importance of a high end soundcard is mainly saved for those hardcore audiophiles who wish for higher audio fidelity than that offered with onboard HD Audio processing.

In 2011 Creative Labs released the first Sound Core3D (ca0132) chipset based soundcards. They featured four Digital Signal Processors (or as Creative Labs like to also call them, Digital Sound Processors) which did all the hardware sound acceleration, offloading that from the Processor, freeing up more cycles for it to use on other tasks, boosting your framerate while maintaining the best audio quality on Windows operating system.

Now... the problem we run into on Linux? Is we aren't running Windows operating system drivers... and the support from Creative Labs for Linux is quite minimal in comparison to Windows. Despite even the hype generated from Valve Software with SteamOS and Steambox? The support for ca0132-based soundcards is still constantly in development. Thanks to advances in the Kernel, Alsa and Pulseaudio? It is possible to get sound input and output to work on ca0132-based soundcards... with some tweaking.

The only "officially" supported implementation of ca0132 on Linux is the Chromebook Pixel.

Easiest way to get sound output to work (kernel v3.16 and up)

  1. Open up "Terminal"
  2. Copy and paste this code into the terminal: 
    sudo kill `ps uax |grep pulseaudio | grep -v grep | awk '{print $2}'` ; sudo rmmod snd-hda-intel; sleep 3; sudo modprobe snd_hda_intel model=,auto
     
    sudo kill `ps uax |grep pulseaudio | grep -v grep | awk '<span class="cm-tag">{print</span> <span class="cm-keyword">$2}</span><span class="cm-string">'` ; sudo rmmod snd-hda-intel; sleep 3; sudo modprobe snd_hda_intel model=,auto</span>
  3. Try playing some audio or a video with audio (for example, a YouTubevideo).
  4. If you have a notebook and no sound is coming out? Open up "Terminal"
  5. Type in "alsamixer" and hit enter
  6. Press "F6" to Select Sound Card
  7. Select "HDA Creative" and hit enter
  8. Use the left and right arrows to navigate the selections and use "Insert" key to enable/disable selections
  9. Select "HP/Speaker Auto Detect" and click your "Insert" key to enable it
  10. Test your sound. If it still doesn't work, try selecting "HP/Speaker" and clicking your "Insert" key to enable it

Easiest way to get sound input (microphone) to work

  1. Open up "Terminal
  2. Type in "alsamixer" and hit enter
  3. Press "F6" to Select Sound Card
  4. Select "HDA Creative" and hit enter
  5. Press "F4" for Capture
  6. Use the left and right arrows to navigate the selections and use "Insert" key to enable/disable selections
  7. Select the "AMic1/DMic Auto Detect" and click your "Insert" key to enable it
  8. If the input is quiet, select "Mic1-Boost" and click your "Insert" key to enable mic boost

Losing sound again on reboot (kernel v3.15 and down)

If you keep losing sound on reboot? We'll need to setup an automatic way of executing these commands again... My example does this and recreates rc.local used in many distros.

  1. Open up "Terminal"
  2. Use this command:
    sudo gedit /etc/systemd/system/rc-local.service
     
    sudo gedit /etc/systemd/system/rc-local.service
  3. Copy & Paste this command set into gedit, save and exit

  

[Unit]
Description=/etc/rc.local Compatibility
ConditionPathExists=/etc/rc.local

[Service]
Type=forking
ExecStart=/etc/rc.local start
TimeoutSec=0
StandardOutput=tty
RemainAfterExit=yes
SysVStartPriority=99

[Install]
WantedBy=multi-user.target

    sudo systemctl enable rc-local.service
  5. Reboot and test if sound works again after restarting Antergos.
